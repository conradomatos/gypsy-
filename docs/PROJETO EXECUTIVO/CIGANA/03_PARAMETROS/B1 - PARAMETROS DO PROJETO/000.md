**Parâmetros do projeto (nível orçamento)** — mudam por obra, definidos no cadastro do orçamento. São as condições específicas:

- Perfil de encargos selecionado (herda do global, pode sobrescrever)
- Local da obra (afeta alimentação, hospedagem, transporte, ICMS)
- Perfil do cliente (VALE/Klabin/genérico — ativa regras)
- Distância casa-obra (afeta horas improdutivas e deslocamento)
- Faturamento direto sim/não
- Fator de dificuldade padrão
- Valores de refeição (café/almoço/jantar — default do global, ajustável)
- Prazo estimado (semanas)
---
**Como funciona na prática:** O orçamentista cria um novo orçamento → sistema puxa todos os defaults globais → orçamentista ajusta o que for diferente nesse projeto específico (ex: "essa obra é no Maranhão, hospedagem é R$ 80/dia e não R$ 150"). O que ele não mexer, fica o global.

É o padrão **"defaults com override"** — parâmetros globais são a base, parâmetros do projeto são a exceção. Cada módulo não tem seus próprios parâmetros — ele consome do projeto, que herda do global.
