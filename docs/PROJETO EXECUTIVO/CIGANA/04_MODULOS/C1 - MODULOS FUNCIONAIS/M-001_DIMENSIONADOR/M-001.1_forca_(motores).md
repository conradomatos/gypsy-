#### FEATURE F-010: Módulo Força — Motores (1.1)

Módulo principal. Tudo nasce da potência dos motores. Dados de placa, tipo de partida, proteção, cálculo de bitola, cabo de comando, terminações automáticas.

**STORY S-010:** Como orçamentista, quero importar a lista de motores do cliente via Excel para não precisar cadastrar motor por motor.
>[!coment] Conrado
>Acho que vale aqui ter cadastrado ja uma lista de motores, la no excel podemos gerar uma planilha com uma lista para escolha, e travar a planilha, para garantir a validacao de dados.

| ----- | ---------------------------------------------------------------------------------------------------------- | ---- | ------- |
| T-100 | Definir schema da tabela dim_motors (potência, tensão, corrente, FP, FS, regime S1-S10, categoria partida) | Task | S       |
| T-101 | Criar migration dim_motors + RLS                                                                           | Task | S       |
| T-102 | Definir template Excel de importação (colunas, validações, exemplo)                                        | Task | S       |
| T-103 | Implementar parser de importação Excel → dim_motors (com validação)                                        | Task | M       |

**STORY S-011:** Como orçamentista, quero que o sistema calcule automaticamente a bitola do cabo de potência para cada motor.

| ID    | Descrição                                                                                                      | Tipo | Esforço |
| ----- | -------------------------------------------------------------------------------------------------------------- | ---- | ------- |
| T-105 | Implementar tabela NBR 5410: capacidade de corrente por bitola × método de instalação × isolação × temperatura | Task | L       |
| T-106 | Implementar cálculo de corrente nominal do motor (In = P / (√3 × V × FP × η))                                  | Task | S       |
| T-107 | Implementar cálculo de corrente de projeto (Ip = In × fator de serviço × fator de agrupamento)                 | Task | S       |
| T-108 | Implementar critério 1: capacidade de corrente (selecionar menor bitola que suporta Ip)                        | Task | M       |
| T-109 | Implementar critério 2: queda de tensão (ΔV ≤ 4% para circuitos terminais, ≤ 7% total)                         | Task | M       |
| T-110 | Implementar critério 3: corrente de curto-circuito (verificar proteção do condutor)                            | Task | M       |
| T-111 | Selecionar maior bitola entre os 3 critérios = bitola final                                                    | Task | S       |
| T-112 | Calcular comprimento do cabo: distância do trecho (F-011) + margem                                             | Task | S       |
>[!coment] Conrado
>Aqui (t-105) é importante tambem, no minimo verificar as tabelas dos fabricantes, principalmente por conta do diametro externo dos cabos. talvez nao seja necessario diferenciar, mas vale revisar. nesse modulo tem calculo critioco, nao pode falhar, aceitar erros, nao pode haver magem para erro aqui, se exisir alguma maneira de validacao de calculos automatico, é aqui que temos que colcoar, nao pode ter erro. se tiver erro precisa mostrar erro, nao pode mostrar resultado errado.
