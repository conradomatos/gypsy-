#### FEATURE F-011: Módulo Infra — Trechos (1.2)

Trechos de infraestrutura compartilhados entre múltiplos alimentadores. Definição manual (decisão humana). Cálculo de ocupação/enchimento e dimensionamento.

**STORY S-020:** Como orçamentista, quero cadastrar os trechos de infra com rota e distância para o sistema calcular a ocupação.

|ID|Descrição|Tipo|Esforço|
|---|---|---|---|
|T-130|Definir schema dim_trechos (nome, ponto_a, ponto_b, distancia_m, tipo_infra, area)|Task|S|
|T-131|Criar migration dim_trechos + RLS|Task|S|
|T-132|Definir schema dim_trecho_alimentadores (trecho_id, motor_id — N:N)|Task|S|
|T-133|UI: cadastro de trechos com nome, pontos, distância, tipo de infra (leito/eletrocalha/eletroduto)|Task|M|
|T-134|UI: associação de alimentadores (motores) a trechos (drag-drop ou multi-select)|Task|M|

>[!coment] Conrado
>Precisa de definições melhores, fazer briefing detalhado sobre. está raso o que tem hoje, especialmente nessa fase de lancamento dos trechos

**STORY S-021:** Como orçamentista, quero que o sistema calcule a ocupação de cada trecho e dimensione o leito/eletrocalha/eletroduto.

| ID    | Descrição                                                                                   | Tipo | Esforço |
| ----- | ------------------------------------------------------------------------------------------- | ---- | ------- |
| T-135 | Implementar cálculo de ocupação: somar seção dos cabos de todos os alimentadores do trecho  | Task | M       |
| T-136 | Implementar regra de enchimento NBR 5410: ≤40% para eletroduto, ≤50% para leito/eletrocalha | Task | S       |
| T-137 | Implementar seleção automática do tamanho da infra (menor que atenda o enchimento)          | Task | M       |
| T-138 | Gerar lista de materiais por trecho: infra + suportes + acessórios + comprimento            | Task | M       |
| T-139 | UI: visualização da ocupação por trecho (barra de enchimento, % usado)                      | Task | S       |
| T-140 | UI: alerta quando trecho excede enchimento máximo                                           | Task | XS      |

>[!coment] Conrado
>Aqui temos que tomar cuidado na hora de gerar os matérias complementares, (suportes, acessorios) pois pode haver trechos que usarao uma solução a e trecho com solução b, exemplo, um trecho precisa de cable hack, outro é so uma mao francesa.

> **Milestone M-003:** Trechos compartilhados calculam ocupação correta com múltiplos alimentadores
