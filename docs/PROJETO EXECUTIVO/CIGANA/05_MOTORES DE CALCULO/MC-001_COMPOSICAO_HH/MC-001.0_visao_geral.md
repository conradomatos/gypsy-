# MC-001 — Motor de Composição de Custo HH

## O que é

Motor de cálculo que transforma salário-base de uma função em custo/hora completo para orçamentação.
É o cálculo mais crítico do Gypsy — todos os custos de mão de obra dependem dele.

## Fórmula core

```
Custo_Funcionario_Mes = Salário_Base
                      + Periculosidade (30% do salário, se aplicável)
                      + Insalubridade (% do salário mínimo, se aplicável)
                      + Encargos_Sociais (sobre remuneração bruta)
                      + Provisão_13º (1/12 da remuneração × (1 + encargos + FGTS))
                      + Provisão_Férias (1/12 × 4/3 da remuneração × (1 + encargos + FGTS))
                      + FGTS (8% da remuneração bruta)
                      + Alimentação (café + almoço + jantar × dias)
                      + Transporte (valor/dia × dias)
                      + EPI (custo mensal por função)
                      + Ferramentas (custo mensal por função)
                      + Exames (custo rateado mensal)
                      + Seguro_Vida (custo mensal)
                      + Hospedagem (se obra fora da base, valor/dia × dias)

Custo_HH = Custo_Funcionario_Mes / Horas_Mes
Custo_HH_com_BDI = Custo_HH / (1 - BDI%)
```

## Inputs

| Input | Tipo | Origem | Exemplo |
|-------|------|--------|---------|
| `funcao` | LaborRole | BD-2 (banco de dados) | Eletricista Oficial |
| `salario_base` | number | BD-2 / tabela salarial | R$ 3.500,00 |
| `periculosidade` | boolean | Parâmetro do projeto | true (30%) |
| `insalubridade_grau` | enum | Parâmetro do projeto | 'nenhum' / 'leve' / 'medio' / 'maximo' |
| `perfil_encargos` | SocialChargeProfile | Parâmetro global | Lucro Real construção civil |
| `alimentacao` | MealCosts | Parâmetro projeto | { cafe: 8, almoco: 25, jantar: 20 } |
| `transporte` | number | Parâmetro projeto | R$ 15/dia |
| `hospedagem` | number | Parâmetro projeto | R$ 0 (local) ou R$ 150/dia |
| `epi_mensal` | number | BD-2 por função | R$ 180 |
| `ferramentas_mensal` | number | BD-2 por função | R$ 50 |
| `horas_mes` | number | Parâmetro global | 220 (mensalista) ou 180 (horista) |
| `dias_uteis_mes` | number | Parâmetro global | 22 |

## Outputs

| Output | Tipo | Descrição |
|--------|------|-----------|
| `custo_hora_sem_bdi` | number | Custo/hora completo sem markup |
| `custo_hora_com_bdi` | number | Custo/hora com BDI aplicado |
| `custo_he_50` | number | Custo hora extra 50% |
| `custo_he_100` | number | Custo hora extra 100% |
| `custo_noturno` | number | Custo hora noturna (20% adicional) |
| `decomposicao` | object | Breakdown de cada componente (para memorial de cálculo) |

## Modalidades de saída

O MC-1 calcula 4 modalidades por função:
1. **Normal** — hora padrão diurna
2. **HE 50%** — hora extra dia útil (custo_hora × 1.5)
3. **HE 100%** — hora extra domingo/feriado (custo_hora × 2.0)
4. **Noturno** — hora noturna (custo_hora × 1.2)

## Dependências

- **MC-002 (BDI)** — fornece o percentual de BDI para calcular preço de venda
- **Parâmetros globais** — perfil de encargos, horas/mês, alimentação default
- **Parâmetros do projeto** — override de alimentação, hospedagem, transporte
- **BD-2 (Banco de MO)** — salário base, EPI, ferramentas por função
