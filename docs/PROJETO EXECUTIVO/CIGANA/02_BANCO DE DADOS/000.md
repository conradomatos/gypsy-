# Banco de Dados — Gypsy

## Source of truth

O schema autoritativo está em **`costai_schema.dbml`** (nesta pasta).
Convenção: inglês 100%, snake_case, UUID PKs, multi-tenant via `organization_id`.

## Domínios

| Domínio | Tabelas | Descrição |
|---------|---------|-----------|
| **System** | `organizations`, `users`, `global_parameters`, `audit_log` | Multi-tenancy, auth, auditoria |
| **Reference (Pricebook)** | `pricebooks`, `materials`, `material_prices`, `labor_roles`, `equipment` | Catálogos mestres. `organization_id = NULL` = dados globais (SINAPI, RSMeans) |
| **Operational** | `budgets`, `budget_items`, `projects`, `clients`, `proposals` | Orçamentos, WBS, propostas. Pertencem ao tenant |
| **Parameters** | `global_parameters`, `project_parameters`, `social_charge_profiles`, `tax_rules` | Configurações empresa e por orçamento |

## RLS

Toda tabela com `organization_id` tem RLS habilitado:

```sql
CREATE POLICY "tenant_isolation" ON [tabela]
  FOR ALL
  USING (organization_id = (SELECT organization_id FROM users
    WHERE auth_user_id = auth.uid()));
```

Dados globais (SINAPI, RSMeans) com `organization_id IS NULL`:

```sql
CREATE POLICY "global_read" ON materials
  FOR SELECT USING (organization_id IS NULL);
```

`audit_log` é INSERT-only: sem UPDATE/DELETE.

## Versionamento de composições

`compositions` usa `composition_root_id` para agrupar versões.
Índice único parcial: `UNIQUE(composition_root_id) WHERE is_current = true`.
Edição cria novo registro com `revision + 1`, marca anterior como `is_current = false`.
Orçamentos referenciam a versão específica (imutabilidade).

## Arquivos deprecados nesta pasta

Os arquivos BD-1 a BD-6 contêm o schema antigo (português, modelo pré-DBML).
**Não usar como referência.** O DBML é a fonte de verdade.
Mantidos temporariamente para rastreabilidade histórica — serão removidos.

## ER resumo

- `organizations` 1:N `users`, `projects`, `budgets`, `materials`, `labor_roles`, `equipment`
- `projects` N:1 `clients` | `projects` 1:N `budgets`
- `budgets` N:1 `bdi_profiles` | `budgets` 1:N `budget_items`
- `budget_items` N:1 `compositions` (versão específica)
- `compositions` 1:N `composition_items` (polimórfico: material, labor, equipment, sub-composition)
- `materials` 1:N `material_prices` (via `pricebooks`)
